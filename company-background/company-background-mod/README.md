# 公司背景（Company Background）

> 负责人：@Dongjianye | 领域：公司级

## 这个 Skill 解决什么问题

团队成员在写产品文案、对外沟通、给新同事介绍公司时，需要准确一致的公司定位和产品描述。如果每个人凭印象写，容易出现数据过时、定位偏差、用语不一致的问题。这个 skill 提供草料的权威背景资料，确保 AI 在涉及公司信息时给出准确、统一的内容。

## 什么时候会触发

当你让 AI 帮你做以下事情时会自动生效：
- 写产品介绍、公司简介、合作方案
- 向客户或合作伙伴介绍草料
- 理解某个产品决策背后的业务逻辑
- 新入职了解公司业务全貌

## 设计考量

内容按"身份 → 愿景 → 文化 → 产品 → 用户价值"的层次组织，从抽象到具体。承重事实与来源对照在 `references/company-facts.md`（每条量化事实带 S1–S5 来源），展开细节在 `references/product-detail.md`，主文件 `SKILL.md` 保持精简编排。事实地基是 `_research/cli-im-sources.md`。

时间敏感数字（用户规模、运营时长）须沿用 `company-facts.md` 的表述与 as-of，不要硬编码会过期的数字。**绝不使用无官方出处的精确数字**（例如坊间的精确生码量"N 亿"说法）。

## 发布闸门与校验

交付/发版前运行：

```bash
node scripts/fact_lint.mjs     # 事实/来源校验（成立年=2011、无无出处精确生码数、关键事实一致、量化事实带来源）
node evals/run_all.mjs         # 可复跑测试：linter 检查 + 行为路由用例
```

发布闸门 = `fact_lint` 全绿 + 与 dossier 一致。负责人 @Dongjianye；时间敏感数字以 S1 表述为准、定期复核。
