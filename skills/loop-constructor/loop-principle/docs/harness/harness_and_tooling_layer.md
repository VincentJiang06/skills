## 1. The five primitives + memory ✓✓

Osmani's "five pieces and then one place to remember stuff" (his taxonomy, not yet a vendor-neutral standard):

1. **Scheduled automations** — fire on a timer, do discovery + triage themselves (`technique.scheduled_automation`).
2. **Git worktrees** — so parallel agents don't step on each other (`technique.git_worktree_isolation`).
3. **Skills** — write down project knowledge so it persists (`concept.skill_as_memory`).
4. **Plugins / connectors** — plug the agent into tools you already use: Jira, GitHub, Slack (`concept.connectors_mcp`).
5. **Sub-agents** — "one has the idea and a different one checks it" (`technique.subagent_split`).
6. **External memory** — "a markdown file, or a Linear board… that lives outside the single conversation and holds what's done and what is next" (`concept.external_state_memory`).

## 2. Hooks — the deterministic glue ✓✓

Claude Code hooks fire shell / HTTP / LLM at lifecycle points — session, turn, and *around each tool call inside the agentic loop* ([hooks](https://code.claude.com/docs/en/hooks)). They turn "please remember to run tests" (advisory, ignorable) into "the turn cannot end until tests pass" (enforced by the harness). A Stop hook with exit code 2 prevents stopping — the mechanism behind `procedure.stop_gate`.

## 3. OpenAI's harness in practice ✓✓

Architectural constraints enforced *mechanically* via (1) custom linters *themselves generated by Codex*, (2) structural tests validating dependency directions, (3) CI jobs blocking boundary-violating PRs; the loop runs until tests/types/builds pass. They wired the **Chrome DevTools Protocol** into the runtime, booting *one app instance per git worktree*, looping *fix → restart → revalidate until the app is clean* (`pattern.cdp_per_worktree_loop`).

> Minor nuance: OpenAI's "Ralph Wiggum Loop" label attaches to their PR-self-review loop, not specifically the CDP UI loop — both are iterative act/check/fix loops.

## 4. Skills as memory — with a security caveat

Anthropic frames capturing successful methods + common errors into reusable context + code (skills) as the loop's knowledge memory. Caveat: skills contain instructions *and code*, can introduce vulnerabilities or exfiltrate data → **install only from trusted sources and audit them** (`anti_pattern.permission_blast_radius`).

Two memory tiers: **knowledge memory** (skills / AGENTS.md / CLAUDE.md / rules) and **state memory** (the external done/next store). The latter is the engineering answer to context drift (`anti_pattern.context_drift`) and the descendant of Reflexion's episodic memory buffer.

See also: `doc.governance.stop_conditions_and_multi_agent` (subagents → maker/checker, worktrees → isolation), `doc.risks.costs_risks_failure_modes` (subagents → token blowup; skills → security).
