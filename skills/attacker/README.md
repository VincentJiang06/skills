# attacker

> 用一个**全新、独立**的攻击者，透过五个由设计哲学推导出的「镜头」去攻击任何目标 —— 只记录已证明、可复现的破坏，永不修复、永不改动目标本身。

[English](README.en.md) · **简体中文**

**做什么** —— 给定任意目标（一个 skill、一份设计、一个论点、一段代码、一整套知识库），派出一个没有任何构建历史的新鲜攻击者，透过**五个镜头的固定轮转**逐一施压；把每一次成功攻击（已定位、可复现、`observed != expected`）写成机器可校验的 **finding**，把没证实的怀疑单独写成 **flag**。输出是一份**交接文档**，**绝不修改目标**。与 [loop-constructor](../loop-constructor/) 配成 攻击→修复→再攻击 的循环。

**这一版（0.5.0）与老版的根本区别** —— 老 attacker 是一套围绕微信小程序/产品调试长出来的、带 `rules/`、`agents/`、多脚本的重装置。这一版是**从设计哲学重新推导**出来的轻量重写：机制被刻意做到极简（fork 一个新脑子 → 给它一个镜头 → 只留它能证明的东西），力量不在机制，而在**递给新鲜大脑的是什么**。整套装置压到老版重量的约 1/4。

**好在哪** ——

- **五镜头 = 最小张成集**。每个镜头对应一条哲学支柱，覆盖一类正交的失败：
  | 镜头 | 问什么 | 支柱 |
  |---|---|---|
  | **Coherence** | 目标自相矛盾吗？（跨条算术、张量仲裁、定义漂移） | P0 一致性 |
  | **Gaming** | 懒惰/作弊者能否字面满足、精神落空？ | A31 / T12 反作弊 |
  | **Evidence** ⚡ | 论断是否真实、时新、如实溯源？（自带联网搜索） | P4 / P5 |
  | **Reality** | 一接触真实目标/真实实现就崩吗？ | P6「部署即认知」 |
  | **Foundation** | 核心前提对吗？会腐烂吗？（攻公理 + 攻演化机制） | 公理 / A41 |
  加第六个镜头是被禁止的，除非它无法折叠进这五个（反臃肿，A41 自反）。

- **工程化的独立性（价值全部来源）**。这个组件针对的缺陷是**假阳性结果** —— 绿色测试套件坐在坏东西上，因为「检查」和「被检查物」出自同一个心智模型（共模错误）。唯一解药是让攻击者从一个**从没见过构建过程**的上下文里跑；高风险时更要从一个**不同的模型**里跑。本版把「换个厂商的模型来当攻击者」提升为一等路径 —— 同族自攻只有 `instance` 级独立性，模型级盲点对它系统性不可见。

- **PROVE-OR-FLAG + SEED 双向门**。PROVE-OR-FLAG 挡假阳性：只有带 `reproduction={steps,expected,observed}`、且能被**非作者的独立复现者**跑出同样破坏，才算 finding；否则落成 flag，绝不粉饰。SEED 挡假阴性：每轮先植入一个已知种子缺陷，漏掉种子的那一轮**作废**、不计入停止条件 —— 防止一个「什么都没找到」的瞎攻击者被误读成「目标干净」。

- **哲学做瞄准，脚本做提取**。当目标是哲学化的知识库时，它的规则里带 lint 强制的「阴影原则 / 可证伪问题」六件套 —— 那就是一张预画好的攻击地图。地图由**确定性脚本**（`scripts/extract_shadow_map.py`）抽取，**绝不用 LLM**（LLM 抽取会重新打开「地图被篡改」的口子）。但**地图是地板不是天花板**：每个镜头至少 30% 预算必须攻击「图外」，而且「阴影原则本身是套话/回避了真正风险」本身就是一类 finding。

- **诚实的停止与覆盖自白**。停止条件是**预注册的预算/边际**门（E9），永远不是「N 轮干净」（对抗电池是渐近的，永远清不完）。输出里的 `coverage_gaps` 如实交代：哪些镜头没跑、达到了哪一级独立性、battery_grade 是不是只到 `smoke-only`。

**什么时候用** —— 「attack this / try to break it」·「red-team 这个想法/设计/方案」·「找出测试漏掉的东西」·「审一遍这套哲学/知识库自身」；也可用 `$attacker` 显式调用，或作为 loop-constructor 设计的循环里的「攻击轮」节点。

**不适用** —— 写/维护项目自己的单元测试、「先写一个失败测试」→ **test-driven-development**（attacker *不信任*那套测试，它攻击运行中的东西）；修复它找到的 bug → 单独的修复轮（attacker 只记录、从不改目标）；设计整个循环 → **loop-constructor**（attacker 只是一个节点）。

**全模型可用（设计约束零，不可谈判）** —— 镜头提示词与评分卡都是 Markdown + 分隔符，不用 XML 语义标签（国内各厂都推 Markdown，且人人能解析）；任何工具 schema 走**六厂交集**（对象根、全 `required`、无 `minLength`/`minItems`、英文 snake_case）；按 128K 安全窗口设计（标称窗口高估，取一半），提示词做成评分卡/清单形（弱指令跟随模型需要显式判据），带 `<think>` 的模型其推理行永不压缩。

**装什么** —— 1 个 `SKILL.md` + 5 个镜头提示词（`lenses/`）+ 1 张评分卡含 ≥12 条黄金样本（`references/prove-or-flag.md`）+ 1 份种子配方（`references/seed-recipes.md`）+ 1 个提取脚本（`scripts/extract_shadow_map.py`，Python 标准库）+ 1 份输出 schema（`schemas/output.json`）。没有 `rules/`，没有按目标定制的脚手架。

**安装** —— `npx skills add VincentJiang06/skills`（或 `cp -R skills/attacker ~/.claude/skills/vince-attacker`）。findings / flags / 台账写在**目标项目**的 `.loop/` 下（项目本地）。

**诚实的覆盖注记** —— 塑造这一版的每一轮攻击都是 `instance` 级（同一 Fable 家族攻击自己的知识库）。按 T11，模型级盲点对它们系统性不可见 —— 所以这里的「收敛」是「对同族攻击收敛」，不是「跨模型验证过」。本 skill 自己的预注册验收测试因此是：**用一个不同厂商的攻击者（GPT / Gemini / DeepSeek / Kimi）对一个真实目标跑一次**，用 SEED 命中率当能力探针。在那次运行存在之前，本 skill 被证明「能找到东西」，但尚未被证明「在真实场景里跨模型可移植」。

完整机制见 [SKILL.md](SKILL.md)。
