# humanizer-academic

> 重写学术文本，去掉 AI 写作痕迹 —— 同时保住学术腔，绝不杜撰事实。

[English](README.en.md) · **简体中文**

**做什么** —— 重写学术 / 学者 / 专业文本（英文、中文，或中英混排），去掉 AI 写作痕迹，同时保留学术腔、绝不杜撰事实。

**好在哪** ——
- 在三个层面去信号 —— **词汇 + 结构 + 统计/突发度**，而不是靠一张词表黑名单。
- 不止做减法：补上定义清晰的**人味纹理**（作者立场、来源具体性、句段长短变化）—— 绝不口语化、绝不编造。
- 随附脚本**只检测、不改写**，永远不是「去 AI 工具」本身。
- 成败交一个**独立盲审**判定，而不是「数我删了几个套路」。

**架构（v4.0.0，模式切分结构重建）** —— 参考文件按「互斥轴」重新切分：模式优先 —— `references/academic-pack.md` / `references/popsci-pack.md`（各自自足）；语言其次 —— `references/lexical-en.md` / `references/lexical-zh.md`；共享非互斥 —— `references/structural-signals.md`；盲审标尺独立成篇。内容**无损吸收**（32/32 覆盖核对，检测脚本与标尺逐字节不变）。实测收益：常驻 SKILL.md 2,868→2,432 token（−15%）；最常见的「判定后不改」路径约 −35%；学术英文重写路径约 −39%（学术任务不再加载科普内容或中文词表）。事实保真护栏同步加固：新增两个反例演练（行为推断、命名实体并列）+ 收紧的第 5 步「零新增事实」扫描 —— 重建过程发现 v3.2.0 自身漏检的一处事实编造（血红蛋白案例），v4.0.0 能抓住。诚实注记：v4.0.0 的赢点在**结构**，质量是「守住而非跃升」—— 盲审 A/B（12 篇）误伤 0、编造 0、去 AI 感 ≥ 基线；学术完成度确有小幅真实提升（+0.25 均分，一篇长文 4→5），科普持平。

**什么时候用** —— 论文章节 / 摘要 / 文献综述 / 研究或政策报告读起来像模板、像 AI 生成，想把它改得像人写、但仍是学术文；也可用 `/humanizer-academic` 显式调用。
**不适用** —— 三个相邻的「假触发」要区分：(1) 口语化的通用 humanizer —— 本 skill **保留学术腔**、不把文字改聊天化；(2) 诗歌 / 演讲 / 小说对白 —— 它们本就该用排比与重复，**别压平**；(3) 只检测、不改写 —— 脚本本身只产出信号图，「只打分别改」的请求只返回检测结果、不做重写。也不用于编造证据 / 引用 / 数字，或非学术的随意文本。
