# Chinese Patterns

> 作者归纳的启发式规则（非来源标注的分类器）。这是中文的**词汇层 SUBTRACT** 目录。
> 结构层与统计层见 `structural-signals.md`；正向 **ADD** 目标见
> `academic-pack.md`。`scripts/detect_ai_signals.py` 将其中一部分家族编码为中文
> 词汇/结构正则层（仅用于诊断）。

中文 AI 味往往不只来自单个词，而是来自整套结构模板、语域模板和段落脚手架。判断时优先看密度和共现，不要只做单词替换。

## 1. 破折号滥用

高风险形式：

- 这不是一次升级——而是一场革命
- 某个普通事实——因此具有重大意义

默认处理：如果破折号只是在制造戏剧感，改成逗号、句号、冒号，或直接改写为事实陈述。只有在插入语、转折、列举承接等确有功能时才保留。

**校准注意**：与英文不同，定量对照（腾讯新闻 2025，7 模型 vs 学生作文基线）显示
中文 AI 的破折号、引号用量**低于**人类；真正超标的是排比/对偶（约 2–6 倍）与
冒号/分号堆叠。所以中文模式下破折号只按"是否在制造戏剧感"判断，绝不按出现
本身扣分——按数量清洗破折号是英文规则的误移植。

## 2. 否定平行 / 对比升维结构（学术模式下默认压缩）

高风险模板（同一家族，全部按此节处理）：

- 不是……而是……（含 并非……而是……、不是……更是……）
- 不仅……还/更……、不只是……更是……、这不仅是……更是……
- 与其说……不如说……
- X 的本质是……、真正的 X 是……、X 从来不是……（"揭示真相"式升维）
- 更重要的是，……不是 A，是 B（口播式转折）

这是当前中文 LLM 最显著的"制造思想感"句式：靠否定一个没人主张的稻草人，
再"揭示"一个升维结论，冒充洞察。

**学术模式默认处理（强制，不是建议）**：默认改写成直接陈述——删掉否定半句，
只保留真正的主张（"不是效率问题，而是激励问题" → "根源在激励结构"）。
**整篇最多保留 1 处**，且仅当源文本确实论证了对比双方（否定侧有真实文献立场
或数据支撑、不是稻草人）时才保留。超过配额 = 改写不合格，回去继续压。
popsci 模式按密度判断（偶发的真对比是合法修辞），但同样不许连用。

## 3. 三连排比与成套框架

高风险形式：

- 三个名词短语并列
- 三层递进副标题
- 首先/其次/最后
- 一方面/另一方面/最后

默认处理：打破过度工整。删掉一项，改一项为例子，或把一项改成事实后果。

## 4. 连接词脚手架

高风险词：

- 此外
- 同时
- 另一方面
- 总的来说
- 综上所述
- 由此可见
- 换言之
- 值得注意的是
- 不可忽视的是

默认处理：如果段落主要靠这些词推进而不是靠内容推进，就删掉脚手架，改用句序和事实衔接。

## 5. 官样、公文、咨询腔

高风险表达：

- 在……背景下
- 随着……不断……
- 围绕……展开
- 对于……而言
- 从……层面看
- 具有重要意义
- 起到重要作用
- 提供有力支撑
- 推动……走深走实

默认处理：不是一律禁用，而是优先压缩空转部分，换成谁做了什么、造成了什么结果。

## 6. 抽象名词化与轻动词

高风险结构：

- 对……进行……
- 开展……建设
- 实现……提升
- 发挥……作用
- 形成……机制
- 构建……体系

默认处理：尽量压回具体动词。

- 对流程进行优化 -> 优化流程
- 起到支撑作用 -> 支撑
- 实现能力提升 -> 提升能力

## 7. 拔高、升华和空泛展望

高风险尾句：

- 意义重大
- 未来可期
- 值得期待
- 书写新篇章
- 注入新动能
- 彰显价值
- 展现魅力

默认处理：把评价改成事实，把升华改成后果，把展望改成下一步。

## 7b. 过度声称与新颖性填充（学术模式）

高风险表达：

- 充分证明 / 有力证明 / 确凿表明（证据只有相关性或单一研究时）
- 首次提出 / 填补了……空白 / 具有开创性意义（源文本没有论证时）
- 大量实验 / 全面系统的研究（没有对应实验规模时）
- 近年来，……受到越来越多的关注 / 随着……的快速发展（模板化开场）

默认处理：**动词强度不得超过证据强度，且只向下修**——把没有支撑的"证明"降为
"表明/提示/与……一致"；绝不把原有的"可能/或许"改强。新颖性口号删掉，留下事实。
模板开场直接换成问题或主张本身。

## 8. 表面分析句

高风险句尾：

- 这说明
- 这体现了
- 这反映出
- 这折射出
- 可见
- 可以说

默认处理：检查后半句有没有新增机制、主体或证据。没有的话，多半应删掉。

## 9. 模糊归因与引用壳

高风险表达：

- 有观点认为
- 专家指出
- 业内普遍认为
- 资料显示
- 多家媒体报道
- 大量研究表明 / 学界普遍认为 / 已有研究指出（后面不接任何具体文献时）

默认处理：源文本里有具体来源就换成具体来源，没有就删。学术模式下还要处理
**引用壳**：每句话结尾机械挂一个括号引文、"根据 XX（2020）的研究"开头套满全段、
"XX（2020）探讨了 Y"这类只报存在不报主张的空转动词——只用**源文本已有的引文**
重排（让被引学者成为主张的主语、把源文本已写明的具体结论说出来），绝不新增、
绝不删除任何引文，绝不替 XX 编造他没说的话。

## 10. 样式级残留

优先清理：

- 粗体标签式列表
- emoji 项目符号
- 编号式聊天回答
- `**2025年：**` / `**增长动力分析：**` 这类粗体领句
- 希望以上对你有帮助
- 作为 AI
- 知识截止说明
- 模板占位符

默认处理：学术文本可以保留必要标题，但不要把整篇文章写成“粗体标签 + 列表条目”的答题格式。

## 11. 报告壳子与元叙述

高风险表达：

- 本文拟……
- 本报告将……
- 下文将从……展开
- 研究背景与意义
- 研究框架与方法
- 增长动力分析
- 主要研究发现

默认处理：这些表达在研究设计部分未必错误，但如果只是占位性开场或报告模板残留，应直接进入事实、问题、方法或结论。

## 12. 章节脚手架过密

高风险形式：

- 第一章 / 第二章 / 第三章 与密集的 1.1 / 1.2 / 2.1 / 2.2 并存
- 每个小节都先来一句“本节将……”
- 一段只有总括句，下一段才给信息

默认处理：保留必要层级，但压缩过密的编号和导语。真正重要的是论证顺序，不是目录感。

## 13. 学术场景说明

学术写作可以保留必要的逻辑连接词，但要保留的是论证关系，不是空话模板。真正该留下的是：

- 然而
- 因此
- 相比之下
- 基于上述结果

真正该删掉的是没有新信息的抬轿句、升华句和脚手架句。

## 14. 白名单

以下文本要降权处理这些规则：

- 诗歌
- 小说对白
- 演讲稿
- 评论性散文
- 文言文 / 古典骈文及其引文（对仗排比是文体本身；市售检测器把《滕王阁序》判
  100% AI、《荷塘月色》判 62.88%，正是把"文体规整"误当"机器生成"的失败模式）

这些体裁本来就可能高频使用破折号、排比、反问和情绪句。另注：§5 的官样词
（赋能/抓手/闭环/底层逻辑）大多是**先于 LLM 存在的**互联网/公文黑话——它们在
学术文本里是"语域混入"信号（该删），但不是 AI 特有指纹，单独出现不构成改写
理由。

## 15. 词汇层之外

清完词汇只是第一步。词汇层之后，要继续处理**结构层**（三连排比、连接词脚手架、报告
壳子元叙述、粗体标签列表、否定平行）和**统计层**（句长/段长过于均匀、对冲均匀铺洒），
见 `structural-signals.md`；然后按 `academic-pack.md` **ADD** 作者立场、
源内具体性与刻意的句长起伏。零命中的稿子仍可能像机器写的。**绝不杜撰任何源文本中
没有的事实、数字、引文或机制来"增加具体性"。**
