# low-visibility-fix

> 审计现场移动端 UI 在低能见度下是否清晰可点 —— 只产出可落地的分级修复文档，绝不动目标本身。

[English](README.en.md) · **简体中文**

**做什么** —— 审计现有的移动端 UI（微信小程序 / H5 / App）在现场低能见度条件（低光、眩光、戴手套、湿手、震动）下是否清晰可点，交回一套可直接落地、按严重度分级的修复方案文档（发现 + 优先级建议）。绝不改目标本身 —— 由另一个 agent 去落地。

**好在哪** ——
- 确定性分析器 + 有界的视觉 / 浏览器走查：阈值由单一来源的设计 token 判定，再用视觉推理补齐长尾。
- 可只锁定具体页面 / 组件，做廉价的多轮复跑 —— 每轮范围可控、扫描有上限。
- 审计与落地清晰分离 —— 本 skill 只产文档（`audit.json` + `report.md`），写入目标源码树会被拒绝。

**什么时候用** —— 「审计这个界面在低光 / 眩光 / 戴手套下是否清晰可点，给我修改建议文档」·「只检查 <某页 / 某组件> 的低能见度问题」；也可用 `/low-visibility-fix` 显式调用。
**不适用** —— 直接改 / 修文件（本 skill 只产文档，由落地 agent 应用）；从零设计新 UI；残障 / 读屏无障碍（色盲、低视力）；与低能见度无关的通用 CSS 审查。

**安装** —— `npx skills add VincentJiang06/skills`（或 `cp -R skills/low-visibility-fix ~/.claude/skills/`）。

完整说明见 [SKILL.md](SKILL.md)。
