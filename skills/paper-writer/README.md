# paper-writer

> 输入一个**论文需求**(字数 / 引用风格 / 章节 / 学科)和/或一个**选题**,从头写出一篇**完整、合规、只引真实来源**的论文。

[English](README.en.md) · **简体中文**

**做什么** —— 把需求 + 选题变成一篇可提交的论文:满足每一条声明的硬约束(长度 / 必需章节 / 引用格式),且**只引可查证的真实来源**。一条编造的引用或抄袭段落不是"差稿",而是学术不端——这个不对称驱动整个设计。

**好在哪** ——
- **两条完整性铁律(常驻上下文,永不违反)**:绝不编造引用/引文/数据(查不到的来源标 `[SOURCE NEEDED]`,绝不发明);绝不抄袭(不把逐字/近逐字的他人文字当原创)。
- **强制的引用存在性核查门(承重设计)**:`scripts/extract_citations.py --verify` 出全量引用清单,逐条必须用真实查证解析为 RESOLVED,否则**用退出码拦截**(草稿伪造不了)——合规报告只有在这道门 exit 0 后才能声称"引用可解析"。因为确定性脚本**只查引用形式、不查存在性**(语法合法的假 DOI 会静默通过),存在性只能靠这道带真实查证的独立核查门。
- **客观 / 主观分叉(C5)**:客观骨架由确定性脚本查(`check_length` / `check_sections` / `check_citations`);主观维度(来源忠实、论证质量、学术腔)由 rubric + 独立判官评。
- **实测**:一次 demo 从「间隔重复/测试效应」选题产出 1323 词 APA7 论文,7 条引用全部**联网查证为真、0 编造**,四道门(结构/字数/引用格式/存在性核查)全过。

**什么时候用** —— 「帮我写一篇关于 X 的论文」·「写一篇…的学术论文」·「把这个需求/选题变成论文」;给定格式规范 + 选题时最佳。

**不适用** —— 润色 / 摘要 / humanize / fact-check 一篇**已有**论文(→ 各自的 sibling skill);无选题无需求的泛写作。

**装什么** —— 1 个 `SKILL.md` + 4 个 `references/`(诚信政策 / 引用风格 / 论文结构 / 主观 rubric)+ 4 个确定性脚本(`scripts/`:长度 / 章节 / 引用格式 / 引用提取+核查门,Python 标准库)+ eval harness。

**诚实注记(v0.1.0)** —— 经 `skill-creator-max` 流水线从头构建(composer→guidance→engineer→独立 battery 验收)。独立 battery 曾抓出一个 P1(诚信核查只是散文、没被强制执行)并驱动修成可运行的带外门。目前 candidate 级:功能已实测可用,但尚未经多轮硬化。

完整机制见 [SKILL.md](SKILL.md)。
