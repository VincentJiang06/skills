# Benchmark Report: vince-skill-conductor

*Subject:* `vinceSKILLcreate/vince-skill-conductor/` · *Date:* 2026-06-04 · *Lens:* class-(A) heavyweight/industrial skill-orchestration only.

## 1. What it does

`vince-skill-conductor` is a **thin orchestrator** that drives one target skill through the full vince-skill pipeline — guidance → engineer → zipper → a final re-audit — with bounded quality-gate loops between stages (`SKILL.md:12-31`). It calls the sibling stage skills, reads the JSON artifact each emits, and runs a machine-readable gate after every stage; a failing artifact is never passed downstream, and the run's real deliverable is a schema-conformant trace at `<target>/.skill-conductor/conductor-log.json` (`SKILL.md:20-21`, `rules/final-acceptance.md:177-208`). It deliberately runs **no domain logic of its own** (no `scripts/` dir; only `SKILL.md`, `rules/`, `assets/`, `evals/`).

## 2. Industrial-grade features (grounded)

- **Run-log JSON contract.** `assets/conductor-log.schema.json` is a strict (`additionalProperties:false`) Draft-2020-12 schema requiring `stages[]`, `loops_taken`, `final_verdict`, and a `quality` object (`schema:8`). Each stage entry pins `stage`/`iteration`/`gate_passed`/`gate_evidence`/`action` enums (`schema:29-55`) — the orchestration is auditable, not narrative.
- **Per-stage gate definitions, expressed as runnable predicates.** `rules/pipeline-loop.md` gives each gate exact pass conditions plus a copy-pasteable `node -e` check (4 embedded gate commands). The G gate verifies the spec parses and carries `recommended_design/prioritized_actions/altitude/scorecard/handoff` + a verdict (`pipeline-loop.md:22-31`). The E gate is **five conjoined criteria** (`pipeline-loop.md:49-98`): verification ran, all required cases pass with `total>0`, every spec-`P0` is `done` (joined by id from the *spec*, not the report — `:91-94`), a **real executable harness the conductor re-runs itself** (rejecting tautological harnesses and hollow red logs — `:60-76`), and an **adversarial-checklist coverage gate** (`:78-89`).
- **Anti-inflation final acceptance.** `rules/final-acceptance.md` makes guidance re-audit the *built* skill, then runs an **independent fresh-subagent behavioral battery** that never saw the build (`:72-107`), adjudicating stdout for green-but-wrong outputs (`:108-113`). The effective verdict is `min(re_audit_verdict, battery_verdict)` and may never exceed the battery (`:124-137`) — pillar-presence scoring caps at `candidate`; `industrial` requires a clean independent battery.
- **Provable termination.** Bounds `MAX_G=2 / MAX_E=3 / MAX_Z=2 / MAX_FULL_LOOPS=3`, with a worst-case arithmetic ceiling of `(MAX_FULL_LOOPS+1)*max=12` attempts/stage (`pipeline-loop.md:176-197`). Budget exhaustion yields an honest `stopped_unmet`, never a relaxed gate (`:199-202`).
- **The skill tests its own contracts.** `evals/round6_pipeline_checks.mjs` (40+ checks) actually *extracts the documented `node -e` gate commands from the markdown and executes them* against synthetic good/bad fixtures (`:79-97, 332-444`), plus a same-target dogfood that clears G/E/Z/final happy-path (`:446-504`), red-log validation, and always-loaded token budgets. **Verified GREEN this session** (conductor SKILL.md = 1068 always-tokens vs 1900 budget).

## 3. Best-in-class external counterparts

- **superpowers** (`learning_from_the_best/superpowers/`, 217k★) — the closest class-(A) analog. A **methodology**: `using-superpowers` forces skill-invocation before any action (`SKILL.md:10-16`); `subagent-driven-development` dispatches a fresh subagent per task with **two-stage review** (spec-compliance *then* code-quality, each looped to approval) and a final whole-implementation reviewer (`SKILL.md:42-87`); `finishing-a-development-branch` adds an environment-detecting merge/PR gate (`:18-39`). Chaining is via prose "Integration" sections + auto-trigger hooks, not a machine contract.
- **anthropics/skill-creator** (`anthropics-skills/skills/skill-creator/`) — sequences draft → **spawn with-skill AND baseline runs in the same turn** → grade against assertions → aggregate a `benchmark.json` with mean±stddev/delta → launch a human review viewer → iterate (`SKILL.md:169-316`). Its standout is **quantitative variance-aware evaluation + a human-in-the-loop viewer**, and a blind A/B comparator (`:325-329`).
- **trailofbits/designing-workflow-skills** (`trailofbits-skills/plugins/workflow-skill-design/`) — the **design rubric** for exactly this skill class: numbered phases with entry/exit criteria, a 5-pattern selector (the conductor is a textbook *Sequential Pipeline*), a 20-entry anti-pattern catalog incl. AP-5 "verify every path resolves" and AP-4 "no hardcoded paths" (`SKILL.md:163-188`).
- **addyosmani planning-and-task-breakdown / incremental-implementation** — vertical-slice decomposition + an implement→test→verify→commit cycle with rationalization tables (class-(A) but single-stage; informative, not orchestrators).

## 4. Head-to-head

**Stronger than the best.** The conductor's **machine-readable gate contract** is something no counterpart has: superpowers chains skills via methodology + auto-trigger hooks but produces no audit artifact and no schema; skill-creator measures quality but has no enforced pass/fail gate that *blocks promotion*. The conductor's `min(re_audit, battery)` anti-inflation rule (`final-acceptance.md:124-137`) and its **closed-loop-breaking independent battery** are a sharper anti-self-certification mechanism than skill-creator's baseline-diff (which the *same* agent still designs). Its **provable finite-loop arithmetic** (`pipeline-loop.md:192-197`) exceeds superpowers' implicit "loop until approved." And its evals **execute the documented commands**, closing the "green-but-wrong docs" gap — stronger eval discipline than any counterpart ships for its own orchestration prose.

**On par.** Independent-reviewer-as-anti-inflation: conductor's fresh-subagent battery ≈ superpowers' two-stage reviewer subagents and skill-creator's blind comparator — same instinct (a verifier that didn't do the work), comparably rigorous.

**Weaker than the best.** (a) **No human checkpoint or quantitative benchmark.** skill-creator's variance analysis (mean±stddev across repeated runs) and review viewer give a richer quality signal than the conductor's single-pass verdict; the conductor is fully autonomous by design but offers no `--review` affordance. (b) **No model-tier routing.** `subagent-driven-development:89-103` selects cheap/standard/capable models per task; the conductor doesn't. (c) **Tooling hygiene per trailofbits.** AP-5 (verify every path resolves) and AP-4 (no hardcoded paths) are violated — see §5. (d) **Operational instructions vs. structure.** superpowers' finishing-a-branch encodes concrete recovery/cleanup commands; the conductor delegates all such mechanics to siblings and stays abstract.

## 5. Gaps & concrete recommendations

1. **STALE PATH (real bug, trailofbits AP-5).** `SKILL.md:36` and `rules/pipeline-loop.md:154` reference the zipper at `../chore-develop-vince-skill-zipper/vince-skill-zipper/…`, but it actually lives at `../vince-skill-zipper/`. Confirmed: the chore-path does **not** resolve; the sibling path does. The eval is green only because `round6_pipeline_checks.mjs:12-19` has a `firstExistingRel` fallback that masks the doc rot. **Fix:** update both doc references to `../vince-skill-zipper/…`. A model literally following the SKILL.md `cp`/`diff_lossless.py` fallback (`pipeline-loop.md:152-155`) would hit a missing file.
2. **STALE KB PATH (off-by-one).** `SKILL.md:38` says `Backing KB: ../develop-principle`, but from `vince-skill-conductor/` that resolves to the non-existent `vinceSKILLcreate/develop-principle`; the real KB is `../../develop-principle` (`skill-developer/develop-principle`). The sibling `vince-skill-guidance/SKILL.md:27` carries the identical off-by-one default — **fix both**, or have the eval assert KB resolvability (it currently doesn't).
3. **Doc rot isn't gated.** The round6 battery asserts token budgets and gate behavior but **not that the paths the prose tells the operator to use resolve**. Add a check that greps `SKILL.md`/`rules/*.md` for sibling/KB paths and asserts each exists (this would have caught #1 and #2) — apply the conductor's own "execute the documented thing" discipline to its path references.
4. **Add an optional human/quantitative checkpoint.** Borrow skill-creator's variance idea: allow an N-run battery with a pass-rate threshold instead of a single fresh-subagent pass, surfacing flaky `industrial` claims. Cheap, high-value, and aligned with the existing battery.
5. **Consider model-tier hints** for sub-stage dispatch (superpowers' pattern) — a note that the fresh-subagent battery and re-audit may use a capable model while mechanical re-runs use a cheap one.

## 6. Verdict

**Yes — industrial-grade by best-in-class standards, and in its core mechanism (an enforced, machine-readable, provably-terminating gate contract with a min(re-audit, independent-battery) anti-inflation rule) it is *ahead* of the named exemplars,** which orchestrate via methodology and measurement rather than a blocking contract. The one thing keeping it from spotless is hygiene, not design: two stale path references (zipper + KB) that its own self-tests don't yet gate — exactly the AP-5 class of bug it is otherwise rigorous about elsewhere.
